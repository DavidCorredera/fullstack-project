### Gestión del Proyecto
Herramienta Utilizada
Para la organización y seguimiento del desarrollo de "Top 5 de Todo", se ha utilizado un tablero Kanban en Trello (enlace disponible en el README principal).

Flujo de Trabajo (Kanban)
El tablero está dividido en las siguientes columnas para gestionar el ciclo de vida de cada tarea:

Backlog: Funcionalidades opcionales, mejoras futuras o ideas que quedan fuera del alcance del MVP actual.

Todo: Tareas pendientes necesarias para cumplir los requisitos del proyecto.

In Progress: Tareas en las que se está trabajando activamente. Se aplica la regla de limitar el WIP (Work In Progress) para mantener el foco.

Review: Tareas finalizadas en código pero que requieren pruebas, revisión de diseño responsive o refactorización antes de darse por válidas.

Done: Tareas completamente terminadas, probadas y funcionales.

Desglose de Tareas
Cada tarjeta en el tablero representa una "Feature" o funcionalidad completa (ej. "Gestión del estado global"). Dentro de cada tarjeta, se utilizan checklists para dividir la funcionalidad en tareas técnicas más pequeñas (ej. "Crear ListContext", "Implementar funciones de LocalStorage"), permitiendo un avance más medible y atómico.
